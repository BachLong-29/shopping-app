# Tailwind CSS Rules

## Styling

- Use utility-first approach
- Avoid excessive class duplication
- Extract reusable variants
- Use consistent spacing scale

## Responsive Design

- Mobile-first design
- Ensure responsive layouts
- Avoid fixed widths when possible

## Class Management

- Use clsx or cn utility
- Keep class order consistent
- Avoid unreadable long class lists

## Theme

- Use CSS variables for theming
- Support dark mode
- Keep color naming semantic
