# Next.js Rules

## App Router

- Use App Router structure
- Prefer Server Components by default
- Use Client Components only when necessary
- Minimize use client usage

## Data Fetching

- Fetch data on the server whenever possible
- Avoid client-side fetching unless needed
- Use caching correctly
- Revalidate strategically

## Routing

- Use route groups for organization
- Keep route structure clean
- Use dynamic routes properly

## SEO

- Use metadata API
- Add proper title and description
- Use semantic HTML
- Optimize Open Graph metadata

## Performance

- Optimize images using next/image
- Lazy load heavy components
- Avoid unnecessary client bundles
- Use streaming when useful

## Security

- Never expose secrets
- Validate server actions input
- Sanitize user-generated content
