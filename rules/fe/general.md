# General Rules

## Core Principles

- Always prioritize readability and maintainability
- Prefer simplicity over clever solutions
- Follow SOLID principles when appropriate
- Avoid premature optimization
- Keep functions pure when possible
- Write self-documenting code

## Code Style

- Use TypeScript everywhere
- Avoid any type
- Avoid deeply nested logic
- Avoid magic numbers
- Extract reusable constants
- Keep files under reasonable size
- Keep components focused on one responsibility

## Imports

- Use absolute imports
- Group imports:
  1. External libraries
  2. Internal modules
  3. Types
  4. Styles

## Error Handling

- Handle all async errors properly
- Never silently ignore errors
- Show user-friendly error messages
- Log important debugging information

## Clean Code

- Remove unused imports
- Remove dead code
- Avoid duplicated logic
- Use meaningful naming
- Keep consistent formatting
